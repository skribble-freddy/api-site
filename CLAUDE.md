## Project Overview

This is the **Skribble API Documentation** site — a Docusaurus v3 site that renders interactive OpenAPI reference docs for Skribble's e-signing platform alongside hand-written guides. It is deployed to `https://api-doc.skribble.com`.

## Commands

```bash
npm run start          # Local dev server (hot reload)
npm run build          # Production build
npm run serve          # Serve the production build locally
npm run clear          # Clear Docusaurus cache (use when builds behave unexpectedly)

# OpenAPI doc generation (fetches specs from external URLs)
npm run gen-api-docs   # Regenerate all API docs from OpenAPI specs
npm run clean-api-docs # Delete all generated API doc files
npm run gen-api-docs:version   # Generate docs for a specific version
npm run clean-api-docs:version # Clean docs for a specific version
```

There are no automated tests in this project.

## Architecture

### OpenAPI Docs (auto-generated)
The three API sections are generated from external OpenAPI YAML specs using `docusaurus-plugin-openapi-docs`:

| Section | Spec source | Output dir |
|---|---|---|
| Sign API v2 | `skribble-freddy.github.io/api-doc/sign_api_v2.yaml` | `docs/sign-api-v2/` |
| Sign API v3 | `skribble-freddy.github.io/api-doc/sign_api_v3.yaml` | `docs/sign-api-v3/` |
| Validation API v1 | `skribble-freddy.github.io/api-doc/validation_api_v1.yaml` | `docs/validation-api-v1/` |

**Do not manually edit files inside `docs/sign-api-v2/`, `docs/sign-api-v3/`, or `docs/validation-api-v1/`** — they are regenerated by `gen-api-docs` and changes will be overwritten. To update those docs, edit the upstream OpenAPI spec, then re-run `npm run gen-api-docs`.

### Hand-written Docs
Narrative guides live at `docs/*.md` and are listed in `sidebars.ts` under the "Infos" category:
- `docs/intro.md` — landing page and quick-start
- `docs/authentication.md`
- `docs/sign-api-best-practices.md`
- `docs/validation-api-best-practices.md`
- `docs/common-workflows.md`
- `docs/error-handling.md`

### Configuration
- `docusaurus.config.ts` — site config, navbar, footer, OpenAPI plugin config, language tabs for code samples
- `sidebars.ts` — sidebar structure; the three generated API sections pull their sidebar from auto-generated `docs/<api>/sidebar.ts` files
- `src/pages/index.js` — homepage
- `src/css/custom.css` — global CSS overrides
